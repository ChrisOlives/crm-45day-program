### Mini Theory (10 min)
Read: HubSpot Workflows — enrollment triggers (contact-based, deal-based, company-based), actions (set property, send email, create task, notify team), and re-enrollment rules. The difference between "Enrolled once" vs. "Re-enrolls when trigger is met again."

### Hands-On Exercise
1. Go to Automation > Workflows > Create Workflow
2. Start a contact-based workflow
3. Set trigger: Contact property "Lifecycle Stage" is "Lead"
4. Add one action: Create a task — "Follow up with new lead"
5. Turn workflow ON (test mode)
6. Manually set a test contact to Lifecycle Stage = Lead — verify task gets created

### Main Project Task
**Build 3 Core BrightClean Automation Workflows**

**Workflow 1: New Lead Notification**
- Trigger: Contact is created AND Lifecycle Stage = Lead
- Actions:
  1. Send internal email notification to Sarah Kim: "New lead: [Contact First Name] [Last Name] — [Lead Source]"
  2. Create task assigned to Sarah: "Call new lead within 2 hours" — due: today + 2 hours
  3. Set Contact property "Lead Status" = New

**Workflow 2: Post-Service Review Request**
- Trigger: Deal Stage moves to "Job Completed"
- Delay: 24 hours
- Actions:
  1. Send email to associated contact (use a simple template): "How did we do? Leave us a review!"
  2. Set Contact property "Review Requested" = True
  3. Create task for Marcus: "Check if review was left in 3 days"

**Workflow 3: Recurring Client Re-booking Reminder**
- Trigger: Contact property "Next Service Date" is exactly 7 days from now
- Actions:
  1. Send internal notification to Operations: "[Contact Name]'s next service is in 7 days — confirm booking"
  2. Create task: "Confirm next appointment with [Contact Name]"

**Test each workflow:**
- Enroll 1 test contact manually into each workflow
- Confirm all actions fire correctly
- Check workflow history/activity log to verify execution
